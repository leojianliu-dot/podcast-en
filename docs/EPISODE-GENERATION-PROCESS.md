# Episode Generation Process

This document describes the end-to-end process for generating a podcast episode, from research to published RSS feed.

## Overview

The podcast is fully generated by Claude Code. The workflow is:

1. **Research** — Search current news from multiple sources and languages
2. **Script** — Write a spoken-word script following the format in `podcast-config.md`
3. **Audio** — Generate MP3 using Edge TTS (Microsoft's free text-to-speech API)
4. **Feed** — Rebuild the RSS feed by scanning the `episodes/` folder
5. **Publish** — Commit and push to GitHub; GitHub Pages serves the files

No manual steps after the user approves the topic direction.

## Directory Structure

```
podcast-en/
├── episode-scripts/          # Markdown scripts (text that gets spoken)
│   └── YYYY-MM-DD-topic.md
├── episodes/                 # Generated MP3 audio files
│   └── YYYY-MM-DD-topic.mp3
├── scripts/                  # Python automation scripts
│   ├── generate_audio.py     # Edge TTS wrapper
│   ├── update_feed.py        # RSS feed generator
│   └── requirements.txt      # Python dependencies
├── docs/                     # Documentation
├── feed.xml                  # Auto-generated RSS feed (do not edit manually)
├── podcast-config.md         # Podcast format, style, and workflow rules
├── podcast-settings.json     # Feed metadata (name, base URL, etc.)
└── topics.json               # Pre-defined news topic categories
```

## Step 1: Research

For daily updates, Claude searches today's news across pre-set topics (see `topics.json`).

For special reports (deep dives), Claude researches a specific topic in depth, pulling from:
- English sources (WSJ, NYT, NPR, Reuters, Bloomberg, CNBC, etc.)
- Regional/international sources (Al Jazeera, Times of Israel, Haaretz, etc.)
- Non-English sources where relevant (Hebrew, Arabic media via web search)

All research is done via web search — no APIs or subscriptions required.

## Step 2: Script Writing

Scripts are saved as Markdown files in `episode-scripts/`.

### File Naming Convention

| Type | Filename | Example |
|------|----------|---------|
| Daily (single) | `YYYY-MM-DD.md` | `2026-02-28.md` |
| Daily morning | `YYYY-MM-DD-morning.md` | `2026-02-28-morning.md` |
| Daily evening | `YYYY-MM-DD-evening.md` | `2026-02-28-evening.md` |
| Special report | `YYYY-MM-DD-topic.md` | `2026-02-28-us-iran-strikes.md` |

### Script Format

- **Target length:** 2,000–3,200 words (10–20 minutes of audio)
- **Special reports** include a `Title:` metadata line at the top (used in RSS feed)
- **Daily updates** get auto-titled based on filename (e.g., "2026-02-28 Daily News")
- See `podcast-config.md` for full format specs and style rules

### Key Style Rules

- Conversational tone — written for the ear, not the eye
- AI disclosure in every opening
- No excessive hedging — be direct
- Sign off with: "I'm Claude. Thanks for listening — catch you next time."

## Step 3: Audio Generation

```bash
python scripts/generate_audio.py episode-scripts/YYYY-MM-DD-topic.md
```

### How It Works

- Uses the `edge-tts` Python library (free, no API key needed)
- Auto-detects language from script content (English vs Chinese)
- Default English voice: `en-US-GuyNeural` (male, professional)
- Output: MP3 file saved to `episodes/` with matching filename
- Typical output: ~4 MB for a 12-minute episode

### Available Voices

| Shortcut | Full Name | Style |
|----------|-----------|-------|
| `guy` | `en-US-GuyNeural` | Male, professional (default) |
| `jenny` | `en-US-JennyNeural` | Female, warm |
| `aria` | `en-US-AriaNeural` | Female, conversational |
| `davis` | `en-US-DavisNeural` | Male, authoritative |
| `andrew` | `en-US-AndrewNeural` | Male, natural |

Override voice: `python scripts/generate_audio.py script.md -v andrew`

### Dependencies

```bash
pip install edge-tts mutagen
```

- `edge-tts` — Text-to-speech generation
- `mutagen` — MP3 metadata/duration reading (optional but recommended)

## Step 4: RSS Feed Update

```bash
python scripts/update_feed.py
```

### How It Works

1. Reads podcast metadata from `podcast-settings.json`
2. Scans `episodes/` for MP3 files
3. For each MP3, reads the matching script in `episode-scripts/` to get:
   - Custom title (from `Title:` metadata line)
   - Description (first few non-header lines)
   - Language detection (English vs Chinese)
4. Gets duration from MP3 metadata (via mutagen) or estimates from file size
5. Generates `feed.xml` — a valid RSS 2.0 feed with iTunes podcast extensions
6. Auto-cleans episodes older than 2 days (configurable)

### Feed URL

The feed is served via GitHub Pages:

```
https://leojianliu-dot.github.io/podcast-en/feed.xml
```

### Auto-Cleanup

`update_feed.py` automatically deletes episodes and scripts older than 2 days (based on filename date, not file modification time). This keeps the repo size manageable since MP3 files are large.

## Step 5: Publish

```bash
git add episode-scripts/ episodes/ feed.xml
git commit -m "Add episode: [title]"
git push origin main
```

GitHub Pages automatically deploys from the `main` branch root. After push:
- Feed is live at: `https://leojianliu-dot.github.io/podcast-en/feed.xml`
- Audio files are served from: `https://leojianliu-dot.github.io/podcast-en/episodes/`

### GitHub Pages Configuration

- **Source:** `main` branch, root `/`
- **Build type:** Legacy (static files, no Jekyll)
- **HTTPS:** Enforced

## Hosting Architecture

```
Local Machine (Claude Code)
    ├── Research (web search)
    ├── Script writing (Markdown)
    ├── Audio generation (Edge TTS → MP3)
    └── Feed generation (Python → XML)
         ↓
    git push
         ↓
GitHub Repository (public)
         ↓
GitHub Pages (static hosting)
    ├── feed.xml          ← Podcast apps subscribe to this
    └── episodes/*.mp3    ← Audio files served directly
```

No servers, no databases, no cloud functions. Just static files on GitHub Pages.

## Subscribing to the Podcast

Add this RSS feed URL to any podcast app:

```
https://leojianliu-dot.github.io/podcast-en/feed.xml
```

Compatible with: Apple Podcasts, Spotify (via RSS), Pocket Casts, Overcast, Google Podcasts, and any other podcast app that accepts RSS feeds.

## Troubleshooting

### Audio generation fails
- Ensure `edge-tts` is installed: `pip install edge-tts`
- Check internet connection (Edge TTS requires network access)
- Try a different voice if one is unavailable

### Feed shows wrong title
- Check the `Title:` line at the top of the script file
- Must be within the first 15 lines of the file
- Format: `Title: Your Episode Title Here`

### GitHub Pages 404
- Verify Pages is enabled: repo Settings > Pages > Source: main branch, root
- Wait 1-2 minutes after push for deployment
- Check that `feed.xml` was committed (not in `.gitignore`)

### Episode not appearing in podcast app
- RSS feeds may cache; wait 15-60 minutes
- Verify `feed.xml` is accessible in a browser
- Check that the MP3 enclosure URL is correct in the feed
